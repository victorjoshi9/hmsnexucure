01Auth & SessionAUTH
Authentication
POST
/auth/login
Login with employee_code + password. Returns access_token (15 min), refresh_token (7 days), employee profile, and role.
PUBLIC
Request body
"employee_code": "DVH-042",
"password": "••••••••",
"device_id": "uuid-xxxx" // Flutter only
Response 200
"access_token": "eyJ...",
"refresh_token": "eyJ...",
"expires_in": 900,
"employee": { id, name, role, dept, shift, photo_url },
"requires_face_enrollment": false
POST
/auth/refresh
Exchange refresh_token for new access_token. Called automatically by Flutter Dio interceptor on 401.
PUBLIC
Request body
"refresh_token": "eyJ..."
POST
/auth/logout
Revoke refresh token. Clears server-side session. Flutter calls this on manual logout.
ALL
POST
/auth/otp/send
Send OTP to registered mobile number. Used when biometric fails or emergency login needed.
PUBLIC
Request body
"mobile": "9876543210"
POST
/auth/otp/verify
Verify OTP and return auth tokens. Skips face check — logged as OTP login in audit.
PUBLIC
Request body
"mobile": "9876543210",
"otp": "847291",
"device_id": "uuid-xxxx"
PUT
/auth/password/change
Change own password. Requires current password. Invalidates all existing refresh tokens.
ALL
Request body
"current_password": "••••",
"new_password": "••••",
"confirm_password": "••••"
POST
/auth/password/reset-request
Admin triggers password reset for an employee. Sends OTP/link to mobile. Admin-only.
HR
Request body
"employee_id": "uuid"
GET
/auth/me
Get own profile from token. Flutter calls on app start to verify session and refresh employee data.
ALL

04Face Recognition & DeviceSECURITY
Face enrollment & device binding
POST
/employees/{id}/face/enroll
Upload 5 face frames for template generation. Server runs ArcFace embedding. Returns quality score per frame and final match confidence. Admin webcam enrollment.
HRmultipart/form-data · 5 images
Response
"template_id": "uuid",
"frames_quality": [98, 97, 95, 99, 96],
"enrollment_score": 97.2,
"status": "enrolled"
POST
/employees/{id}/face/verify
Verify a single live face frame against stored template. Called by Flutter during check-in/break before submitting attendance. Returns match score.
ALLmultipart · 1 image + liveness data
Request (multipart)
image: binary frame
"liveness_blink": true,
"liveness_head_turn": true
Response
"match": true, "score": 97.4,
"liveness_passed": true,
"verify_token": "short-lived-uuid" // pass to check-in
DELETE
/employees/{id}/face
Delete face template. Employee must re-enroll before next check-in. SA/HR only.
HR
POST
/employees/{id}/device/bind
Bind a specific device_id to employee. Called automatically on first Flutter login. Admin can force-bind from panel.
HR
Request body
"device_id": "uuid-xxxx",
"device_model": "Samsung Galaxy A54",
"os_version": "Android 14"
DELETE
/employees/{id}/device
Reset device binding. Employee must log in on new device. Admin does this when phone is lost/replaced.

Request body

/attendance/{attendance_id}/breaks/{break_id}/end
End active break. Requires face verify_token. Computes duration, flags violation if over limit. Updates status to "working".

Request body

Response

"is_violation": true,
"excess_minutes": 8,
"total_break_today": 83

WebSocket — SignalR hub
GET
/hubs/attendance
SignalR WebSocket hub. Admin panel connects for real-time live monitor. Flutter connects for real-time status updates on home screen.
ALL
Client subscribes to events
OnCheckIn — { employee_id, name, time, status }
OnCheckOut — { employee_id, name, time, working_hours }
OnBreakStart — { employee_id, break_type, time }
OnBreakEnd — { employee_id, duration_min, is_violation }
OnAbsenceAlert — { employee_id, shift_ended_at }
OnCorrectionRequest — { request_id, employee_name, type } // admin only
Standard error shape:
{ "error": { "code": "FACE_MISMATCH", "message": "Face match score 72% below threshold 95%", "field": null } }

Common error codes: UNAUTHORIZED · FORBIDDEN · DEVICE_NOT_BOUND · FACE_MISMATCH · LIVENESS_FAILED · OUTSIDE_GEOFENCE · MOCK_GPS_DETECTED · ALREADY_CHECKED_IN · NO_ACTIVE_BREAK · BREAK_ALREADY_ACTIVE · LEAVE_BALANCE_EXCEEDED · SHIFT_NOT_ASSIGNED
