Inline edit rules

Any manual edit requires: Reason (text field, min 20 chars) + HR password re-confirm. Edit is saved with edited_by, edited_at, and edit_reason written to AuditLogs. Original value preserved in original_value column. Edits visible to Super Admin only in audit view.

Break log sub-view

Click any row to expand a break log showing all break events for that attendance record: break type, start, end, duration, violation flag.

06Attendance CorrectionsWORKFLOW

Two-stage approval workflow. Employee submits via app. Manager reviews, HR finalizes. Full audit trail.

Correction request types

Forgot check-in
Employee provides reasonRequested check-in timeSupporting photo (optional)
Forgot check-out
Employee provides reasonRequested check-out timeManager can verify via CCTV note
Location issue
GPS failed / network errorEmployee-provided coordinatesAdmin verifies manually
Wrong status
System marked incorrectlyCorrect status requestedHR-only approval

Approval workflow

1
Employee submits via app
Request logged with timestamp, request type, reason, requested value. Status → Pending.
2
Dept Head review (optional)
If configured, dept head gets notified. They can approve/reject/comment. Status → Dept Approved.
3
HR / Super Admin final approval
HR sees approved corrections queue. Reviews + approves → attendance record updated. Status → Resolved. Audit log entry created.

10ReportsSCREEN

Seven report types. All generated server-side. PDF rendered by QuestPDF, Excel by ClosedXML. Large reports queued as background jobs (Hangfire); download link sent via email when ready.

Daily attendance
All staff for one date. Present / absent / late / break violations. PDF or Excel. ↗
Monthly staff report
Per-employee summary for a month. Days present, absent, late, OT hours. ↗
Department report
Attendance % by dept, late analysis, staff utilization chart. ↗
Late arrivals
All late punches in date range with minutes late and frequency ranking. ↗
Overtime report
OT hours per employee, OT cost estimate based on salary config. ↗
Break violations
Employees exceeding break limits, total excess time, deduction impact. ↗
Leave summary
Leave taken per type per employee, balance remaining, encashment eligible. ↗
Scheduled reports: Admin can configure daily / weekly / monthly reports to auto-generate and email to configured recipients. Hangfire cron job, configurable from Settings > Reports.

absent_daysNUMBERAbsent without approved leave
late_countNUMBERNumber of late arrivals — for deduction rule if configured
total_late_minutesNUMBERTotal late minutes in month
total_working_hoursNUMBERSum of net working hours after breaks
overtime_hoursNUMBERTotal OT hours (triggers OT pay if salary_rules configured)
leaves_takenJSONPer-type breakdown: CL: 2, SL: 0, EL: 1
half_daysNUMBERCount of half-day attendances
break_excess_minutesNUMBERTotal excess break time (if to be deducted)

Export formats

Excel (.xlsx) — full detail CSV — for salary software import PDF — printable payroll summary Custom column mapping (match target software's column names)

Auth header: All admin endpoints require Authorization: Bearer <JWT>. Token expiry: 15 min access token + 7-day refresh token. Role claim in JWT payload used for server-side RBAC on every route. Rate limit: 300 req/min per IP on admin endpoints.
